 HM-10 is one of the most famous BLE module for DIY. Core device is TI's SoC CC2540/2541 (both type is available), with SPP firmware. Also you can re-programming/write your custom firmware into MCU, for more detail: [CC254X development].

(from blog.blecentral.com) there's some types of fake/clone model for HM-10. If you don't want to catch them, check appearance and also confirm to the seller before you raise an order. According to the document bluetooth40_en.pdf,

A lot of fake modules in the market Such as HC-08 and CC41-A is the copycat company copied from our products, If you buy a fake, please apply for a refund guarantee your legitimate rights and interests.

There are many types of BT modules comes from Huamao Technology, now mainly available model is HM-10, HM-11. The smaller number model HM-N (N<=9) is maybe discontinued (supports only Bluetooth V2.1 (+EDR)), and the larger number model HM-N (N>=12) is newer item but a bit expensive or poor availability, also fewer information on the web. (HM-14 is no longer available. HM-15 is USB dongle.) The factory firmware AT command instruction is basically compatible with each other module.

 JDY-10 is one of the clone model for HM-11. There is no AT command to change role (master/slave), it means this module (firmware) supports only slave mode. Some seller described the chip is CC2541, but actually it uses TLSR8266. Since "TLSR8266" has no program flash (ROM), and chip silk can be read ...2ET..., then its package is 32pin TQFN, so finally I think it is TLSR8266F512ET32. I don't know how to develop this chip or re-programmable via PCB pinouted line. TL;DR: As compare with JDY-08, MCU is more powerful, however the firmware function is poor, the document or information is fewer.
